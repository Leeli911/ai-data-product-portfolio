# Review Checklist

每次提交前检查以下内容。

## 代码

1. 是否能运行。
2. 是否有 `requirements.txt`。
3. 是否有中文注释。
4. 是否避免复杂依赖。
5. 是否没有无用文件。
6. 是否没有把临时调试代码提交进来。

## 测试

1. 是否新增或更新了单元测试。
2. 是否使用 `pytest` 执行测试。
3. 是否记录了测试命令和结果。
4. 测试是否写清楚输入和预期输出。
5. 测试是否覆盖核心业务逻辑。
6. 如果没有运行测试，是否说明原因。

## 产品

1. README 是否说明业务场景。
2. 是否说明 AI workflow。
3. 是否说明评测指标。
4. 是否有项目价值说明。
5. 是否体现 AI 数据产品经理能力。
6. 是否避免写成内部练习说明。

## 数据

1. mock 数据字段是否清楚。
2. 指标含义是否明确。
3. 诊断结论是否由数据支持。
4. 环比、变化率和主因分析是否合理。
5. 是否避免过度推断。

## GitHub 展示

1. 根目录 README 是否能在一分钟内讲清项目价值。
2. 目录结构是否清晰。
3. 文档命名是否统一。
4. 是否适合作为简历链接。
