# Skill: NL2SQL Workflow

## 适用场景

当任务涉及智能问数、自然语言取数、SQL 生成、业务分析诊断时，使用本技能。

## 标准流程

1. 识别用户问题中的指标。
2. 识别时间范围。
3. 识别地区、业务线等维度。
4. 生成结构化 query。
5. 根据结构化 query 生成 SQL。
6. 使用 mock 数据做分析。
7. 输出业务诊断说明。

## 输出要求

每次实现该流程时，至少包含：

1. Parsed Intent
2. Generated SQL
3. Analysis Result
4. Business Diagnosis

## 产品解释

该流程用于模拟 AI 数据产品中的智能问数能力。

LLM 不直接负责计算，计算应交给数据库或 pandas。LLM 主要负责语义理解、结构化生成和结果解释。

在本作品集中，第一阶段不接真实大模型 API，先用规则和模板模拟 AI workflow。这样可以更清楚地展示产品经理对流程、边界和评测的理解。

## TDD 要求

实现 NL2SQL workflow 时，应先写测试再写实现。

测试至少覆盖：

1. 中文问题能否解析成结构化 intent。
2. intent 能否生成字段和条件正确的 SQL。
3. SQL 文本是否包含目标指标、目标表和目标筛选条件。
4. 异常问题是否有合理默认值或边界说明。
